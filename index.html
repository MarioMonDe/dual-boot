Descargar Ventoy (grabador de USB)

Ventoy es la herramienta más cómoda: metes varias ISOs en un solo USB y elige en el arranque cuál usar.

Sitio oficial
https://www.ventoy.net/en/download.html
✦

Ventoy permite usar un solo USB para ambas ISOs. Simplemente copia los dos archivos .iso a la partición de datos del USB después de instalarlo.

4
Preparar el USB con Ventoy (desde Windows o Linux actual)

Ejecuta Ventoy2Disk, selecciona tu USB y haz clic en Install. Luego copia las ISOs:

USB raíz/
├── ubuntu-26.04-desktop-amd64.iso
└── Parrot-security-6.x_amd64.iso
5
Deshabilitar Secure Boot en la BIOS/UEFI

Reinicia → pulsa Supr o F2 (varía por placa) al arrancar para entrar en UEFI.

Navega a: Security → Secure Boot → Disabled

Guarda con F10 y reinicia.

⚠

Con un Ryzen 7 3700X la placa base suele ser MSI o ASUS. Busca "Secure Boot" dentro de la pestaña Boot o Security de tu UEFI.

Instalar Ubuntu 26.04 LTS

Primera instalación — configura el bootloader GRUB

1
Arrancar desde el USB

Conecta el USB Ventoy → reinicia → pulsa F11 o F12 (Boot Menu) → selecciona el USB → selecciona la ISO de Ubuntu 26.04.

2
Iniciar el instalador de Ubuntu

En el menú Ventoy elige ubuntu-26.04-desktop-amd64.iso. Al arrancar verás el logo de Ubuntu y después el instalador "Resolute Raccoon".

Selecciona idioma: Español → clic en Instalar Ubuntu.

3
Configuración inicial del instalador
OpciónValor a elegir
TecladoEspañol (o el tuyo)
Tipo de instalación de appsInstalación predeterminada
Instalar software de tercerosSí (drivers NVIDIA RTX 2060)
Conectar a internetSí (actualizaciones automáticas)
4
Tipo de instalación — IMPORTANTE

Cuando el instalador pregunte "¿Cómo quieres instalar Ubuntu?", elige:

▸ "Instalación manual" / "Algo más" — NO elijas "borrar disco".

🔴

Si eliges "Borrar disco e instalar Ubuntu" eliminarás todo tu disco. Siempre elige instalación manual para dual boot.

5
Crear las particiones manualmente

En el editor de particiones del instalador:

Paso 5a: Crea la partición EFI (si no existe ya):

Tamaño: 1024 MB
Tipo: Partición primaria
Usar como: FAT32
Punto de montaje: /boot/efi
☑ Marcar como "Partición de arranque EFI"

Paso 5b: Crea la partición raíz de Ubuntu:

Tamaño: 153600 MB  (= 150 GB)
Tipo: Partición primaria
Usar como: Sistema de archivos ext4
Punto de montaje: /

Paso 5c: Crea el espacio libre para Parrot (déjalo sin formatear ahora):

Tamaño: 100000 MB  (= ~100 GB)
Tipo: dejar como "espacio libre"  ← Parrot lo usará después

Paso 5d: Crea la partición de swap:

Tamaño: 32768 MB  (= 32 GB)
Tipo: Partición primaria
Usar como: Área de intercambio (swap)
6
Dispositivo para el cargador de arranque

En el selector de bootloader, elige el disco principal (no una partición concreta):

/dev/sda  ←  el disco completo, NO /dev/sda1 ni /dev/sda2
7
Completar la instalación

Configura: zona horaria, nombre de usuario y contraseña. Haz clic en Instalar ahora. La instalación tarda entre 10 y 20 minutos.

Al terminar: reiniciar → retirar el USB cuando lo pida.

✦

Ubuntu arrancará correctamente con GRUB. Verifica que todo funciona antes de instalar Parrot.

03

Instalar Parrot Security OS

Segunda instalación — en el espacio libre reservado

1
Arrancar desde el USB en Parrot

Reinicia → Boot Menu (F11/F12) → USB Ventoy → selecciona la ISO de Parrot Security.

En el menú de Parrot elige: Try/Install → espera que cargue el live desktop.

2
Abrir el instalador

En el escritorio live haz doble clic en el icono "Install Parrot" (Calamares installer).

3
Configuración inicial Parrot
OpciónValor
IdiomaEspañol
Zona horariaEuropa/Madrid (Canarias → Atlántico/Canarias)
TecladoEspañol
4
Particionado — usar el espacio libre reservado

Elige "Particionado manual". Verás el espacio libre de ~100 GB que dejaste antes.

Selecciona ese espacio libre → Crear nueva partición:

Tamaño: todo el espacio libre (~100 GB)
Sistema de archivos: ext4
Punto de montaje: /
Marca: Sin bandera (no marques como boot)
ℹ

La partición /boot/efi ya existe (la creó Ubuntu). En el instalador de Parrot, asígnala como /boot/efi sin formatear (opción "mantener").

5
Asignar la partición EFI existente

Selecciona /dev/sda1 (la partición FAT32 de 1 GB que creó Ubuntu):

Sistema de archivos: Mantener (No formatear)
Punto de montaje: /boot/efi
🔴

NO formatees /boot/efi. Si la formateas borrarás el GRUB de Ubuntu y tendrás que repararlo.

6
Crear usuario en Parrot

Elige un nombre de usuario diferente al de Ubuntu para evitar confusiones (por ejemplo mario-parrot).

Contraseña robusta: mínimo 12 caracteres, mezcla de mayúsculas, números y símbolos.

7
Finalizar instalación Parrot

Clic en Instalar. Tarda unos 15 minutos. Al terminar, reinicia y extrae el USB.

GRUB de Parrot detectará Ubuntu automáticamente. Verás ambas opciones al arrancar.

04

Verificar y ajustar GRUB

Asegurarse de que ambos sistemas aparecen en el arranque

1
Verificar entradas de GRUB desde Parrot

Arranca en Parrot y abre una terminal:

TERMINAL
sudo update-grub

Deberías ver líneas que detectan Ubuntu 26.04 automáticamente.

2
Instalar os-prober si no detecta Ubuntu
sudo apt install os-prober
sudo os-prober
sudo update-grub
3
Cambiar el sistema por defecto del GRUB (opcional)

Por defecto arranca Parrot. Para cambiar a Ubuntu como opción predeterminada:

sudo nano /etc/default/grub

# Cambia esta línea (0 = primera entrada, normalmente Parrot):
GRUB_DEFAULT=2   # ajusta el número según la posición de Ubuntu

# Guarda con Ctrl+O, Ctrl+X
sudo update-grub
4
Aumentar tiempo de espera del GRUB
sudo nano /etc/default/grub

GRUB_TIMEOUT=10   # 10 segundos para elegir

sudo update-grub

Post-instalación Ubuntu 26.04

Configuración para desarrollo Flutter · Android · Python

1
Actualizar el sistema completo
sudo apt update && sudo apt upgrade -y
sudo apt autoremove -y
2
Instalar driver NVIDIA (RTX 2060)

Ubuntu 26.04 detecta la RTX 2060 automáticamente si marcaste "instalar software de terceros". Verifica:

nvidia-smi

Si no aparece, instala manualmente:

sudo ubuntu-drivers autoinstall
sudo reboot
3
Instalar herramientas de desarrollo base
sudo apt install -y \
  git curl wget unzip build-essential \
  apt-transport-https ca-certificates \
  gnupg lsb-release software-properties-common
4
Instalar Flutter + Dart
# Instalar snapd (ya incluido en Ubuntu)
sudo snap install flutter --classic

# Verificar instalación
flutter doctor

# Aceptar licencias Android
flutter doctor --android-licenses
5
Instalar Android Studio
sudo snap install android-studio --classic

# O descarga desde:
# https://developer.android.com/studio
6
Instalar Python vía pyenv
# Instalar dependencias
sudo apt install -y libssl-dev zlib1g-dev libbz2-dev \
  libreadline-dev libsqlite3-dev libffi-dev

# Instalar pyenv
curl https://pyenv.run | bash

# Añadir al perfil (~/.bashrc o ~/.zshrc)
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc
source ~/.bashrc

# Instalar Python 3.12
pyenv install 3.12
pyenv global 3.12
7
Instalar VS Code
sudo snap install code --classic
8
Instalar Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
newgrp docker
9
Instalar Fish shell + terminal moderna
sudo apt install -y fish
chsh -s $(which fish)   # establecer como shell por defecto

# Instalar Fisher (gestor de plugins de Fish)
fish -c "curl -sL https://raw.githubusercontent.com/jorgebucaran/fisher/main/functions/fisher.fish | source && fisher install jorgebucaran/fisher"

Configuración para Hack4u / S4vitar

Entorno de trabajo tal como lo usa S4vitar en sus cursos

1
Instalar zsh con powerlevel10k (shell de S4vitar)
sudo apt install -y zsh
chsh -s $(which zsh)

# Instalar Oh My Zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# Instalar powerlevel10k
git clone --depth=1 https://github.com/romkatv/powerlevel10k.git \
  ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/powerlevel10k

# En ~/.zshrc cambiar:
ZSH_THEME="powerlevel10k/powerlevel10k"

# Reiniciar terminal y seguir el asistente de configuración
2
Instalar plugins de zsh esenciales
# zsh-autosuggestions
git clone https://github.com/zsh-users/zsh-autosuggestions \
  ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

# zsh-syntax-highlighting
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git \
  ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

# Activar en ~/.zshrc:
plugins=(git zsh-autosuggestions zsh-syntax-highlighting)
3
Instalar fzf (búsqueda fuzzy de comandos)
git clone --depth 1 https://github.com/junegunn/fzf.git ~/.fzf
~/.fzf/install
✦

S4vitar usa mucho Ctrl+R con fzf para buscar comandos anteriores. Imprescindible en su workflow.

4
Instalar bat, lsd y otros reemplazos modernos
# bat (reemplaza cat con colores)
sudo apt install -y bat
# En algunos sistemas el binario se llama batcat:
echo "alias bat='batcat'" >> ~/.zshrc

# lsd (reemplaza ls con iconos)
sudo apt install -y lsd
echo "alias ls='lsd'" >> ~/.zshrc
echo "alias ll='lsd -la'" >> ~/.zshrc

source ~/.zshrc
5
Configurar aliases de pentesting (estilo S4vitar)
# Añadir al final de ~/.zshrc

# Escaneo rápido con nmap
alias escaneo="nmap -sC -sV -oN targeted"
alias descubrimiento="nmap -sn"

# Directorio de trabajo para CTF
alias ctf="cd ~/CTF && ls"

# Abrir Burp Suite rápido
alias burp="burpsuite &"

source ~/.zshrc
6
Estructura de carpetas para CTF (estilo S4vitar)
mkdir -p ~/CTF/{HackTheBox,TryHackMe,Hack4u}
mkdir -p ~/CTF/HackTheBox/Maquinas
mkdir -p ~/Tools

# Estructura de cada máquina:
mkdir -p ~/CTF/HackTheBox/Maquinas/NombreMaquina/{nmap,content,exploits}
7
Instalar kitty terminal (la que usa S4vitar)
sudo apt install -y kitty

# Configurar kitty (~/.config/kitty/kitty.conf)
mkdir -p ~/.config/kitty
cat > ~/.config/kitty/kitty.conf << 'EOF'
font_family      JetBrains Mono
font_size        13.0
background_opacity 0.9
cursor_shape     block
enable_audio_bell no
EOF
